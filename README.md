# RemnaWave Management API

Backend-сервис для управления VPN клиентами в RemnaWave с полным аудитом операций и автоматической деактивацией просроченных подписок.

## 📋 Обзор проекта

Реализован полнофункциональный backend-сервис для управления VPN клиентами, соответствующий всем требованиям технического задания и включающий дополнительные улучшения для продакшен использования.

## ✅ Выполненные требования

### Основной API:
- ✅ **Управление клиентами**: создание, получение списка, удаление
- ✅ **Подписки**: продление на N дней
- ✅ **Доступ**: блокировка/разблокировка
- ✅ **Конфигурации**: получение и перевыпуск конфигов

### Плюс-функции:
- ✅ **Аудит операций**: все действия логируются в таблице `operations`
- ✅ **Авто-деактивация**: фоновая задача блокирует просроченных клиентов
- ✅ **Идемпотентность**: защита от дублей в ключевых операциях

### Технические требования:
- ✅ **PostgreSQL**: основная база данных
- ✅ **Docker Compose**: запуск одной командой
- ✅ **Валидация данных**: все ошибки в JSON формате с HTTP кодами
- ✅ **Unit тесты**: 10 тестов на бизнес-логику

## Технический стек

- **Backend**: FastAPI (Python 3.11)
- **База данных**: PostgreSQL
- **Оркестрация**: Docker Compose
- **Тестирование**: pytest
- **Аудит**: SQLAlchemy + кастомный сервис

## Модель данных

### Client
```sql
id (UUID) - уникальный идентификатор
status (active/blocked) - статус клиента
expiresAt (timestamp) - дата окончания подписки
createdAt (timestamp) - дата создания
updatedAt (timestamp) - дата обновления
```

### Operations (Аудит)
```sql
id (UUID) - уникальный идентификатор операции
clientId (UUID) - ID клиента
action (string) - тип операции (create, extend, block, unblock, delete, config_rotate, auto_deactivate)
payload (json) - параметры операции
result (success/fail) - результат операции
error (text) - текст ошибки (если есть)
createdAt (timestamp) - дата операции
```

## API Эндпоинты

### Клиенты
- `POST /clients` - создать клиента
- `GET /clients` - список клиентов с фильтрами
- `GET /clients/{id}` - карточка клиента
- `DELETE /clients/{id}` - удалить/деактивировать клиента

### Подписка
- `POST /clients/{id}/extend` - продлить на N дней

### Доступ
- `POST /clients/{id}/block` - заблокировать
- `POST /clients/{id}/unblock` - разблокировать

### Конфигурация
- `GET /clients/{id}/config` - получить конфигурацию
- `POST /clients/{id}/config/rotate` - перевыпустить конфиг

### Аудит
- `GET /operations?clientId=...` - получить операции

```bash
docker-compose exec api pytest tests/ -v
```

## 📹 Демонстрация

Скрипт `demo.py` показывает полный рабочий процесс:

1. **Создание клиента** - новый клиент с подпиской на 30 дней
2. **Получение информации** - детальная информация о клиенте
3. **Конфигурация** - получение VPN конфигурации
4. **Продление подписки** - добавление 15 дней
5. **Блокировка** - отключение доступа
6. **Разблокировка** - восстановление доступа
7. **Ротация конфига** - перевыпуск ключа
8. **Просмотр аудит лога** - все операции клиента
9. **Удаление клиента** - полная очистка

## Примеры использования

### Создание клиента
```bash
curl -X POST "http://localhost:8000/clients" \
     -H "Content-Type: application/json" \
     -d '{"days": 30}'
```

### Получение конфигурации
```bash
curl -X GET "http://localhost:8000/clients/{client_id}/config"
```

### Продление подписки
```bash
curl -X POST "http://localhost:8000/clients/{client_id}/extend" \
     -H "Content-Type: application/json" \
     -d '{"days": 15}'
```

### Блокировка клиента
```bash
curl -X POST "http://localhost:8000/clients/{client_id}/block"
```

## Особенности реализации

### Идемпотентность
- Операции создания и продления защищены от дублирования
- Повторные запросы с теми же параметрами не создают дубликатов

### Авто-деактивация
- Фоновая задача проверяет клиентов каждые 5 минут
- Просроченные клиенты автоматически блокируются в RemnaWave и в БД
- Все действия логируются в аудит

### Обработка ошибок
- Все ошибки возвращаются в стандартизированном JSON формате
- Детальное логирование для отладки
- Graceful degradation при недоступности RemnaWave

### Тестирование
- Unit тесты на бизнес-логику
- Тесты планировщика и аудита
- Изолированная тестовая база данных
